## Practiced product path

The final video is generated by Understand Video, not assembled as a slide-only
explainer. Its render performs these visible and machine-checked steps:

1. validate the clean TEMPO snapshot and the reviewed graph identity;
2. resolve every claim to the exact source range above;
3. render source, architecture, authority, lease, and audit scenes;
4. execute the credential-free Understand Video judge bundle offline;
5. display its real terminal output and require exactly one
   `UNDERSTAND_VIDEO_DEMO_PASSED` sentinel; and
6. verify MP4, audio, captions, safe zones, grounding, and receipts.

## TEMPO mechanism contract

The pinned TEMPO checkout must pass:

```powershell
python bin/tempo demo
python bin/tempo selfcheck
python bin/tempo verify
python bin/tempo ledger verify
```

The video distinguishes a signed warrant from a live build lease. A valid
warrant is necessary but not sufficient; `mvp start` revalidates authority and
creates one exact lease. Commit `4a73350` permits an atomic rotation only for
the same actor, session, task, warrant, lane, and authorized action boundary.
Cross-owner, cross-task, or out-of-scope starts remain fail-closed.

## Claim boundaries

- TEMPO is a hackathon vertical slice, not a production authorization service.
- Local commands are not a hostile-code sandbox; local ledger and receipts are
  local-integrity evidence, not external notarization.
- Demo fixtures prove mechanisms, not customer demand or market impact.
- The current TEMPO runtime does not call the OpenAI API. GPT-5.6 is accurately
  described as materially used through Codex during the build.
- No measured time or cost savings are claimed.
